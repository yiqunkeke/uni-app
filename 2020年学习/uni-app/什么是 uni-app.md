## 什么是 uni-app

- uni-app 是一个使用 vue.js 开发所有前端应用的框架，开发者编写一套代码，可发布到 ios，andriod，h5以及各种小程序（微信/支付宝/百度/头条）、快应用等多个平台。

- DCloud 公司拥有500万开发者用户，几十万应用案例、10亿手机端月活用户。

  阿里小程序工具官方内置uni-app。

  开发者可以放心选择。

- uni-app在手，做啥都不愁。即使不跨端，uni-app 也是更好的小程序开发框架

- 如果你需要工程化能力，那就直接忘了微信原生开发吧。

- 跨端框架，还涉及一个ui框架的跨端问题：

  uni-app官方提供了**uni ui**，可全端运行。

  uni-app还有一个插件市场，里面有很多**三方ui组件**。

  

## 为什么选择uni-app

- 学习成本低：**采用 vue 语法 + 微信小程序 api**，无额外学习成本
- 跨平台能力：支持**条件编译**+平台特有api调用，可以为某平台写个性化代码，调用专有能力而不影响其他平台。

- 功能框架：uni-app内置组件和api。uni-app 将常用的**组件和 api进行了跨平台封装**，可覆盖大部分的业务需求。

## DCloud

- DCloud 是小程序的开创者。

- 2012年，开始研发小程序技术。优化webview的功能和性能。加入W3C和HTML5中国产业联盟。推出HBuilder开发工具。

- 2015年，DCloud正式商用了自己的小程序，产品名为“流应用”。它不是 B/S 模式的轻应用。而是能接近原生功能、性能的动态App，并且即点即用。

- 2015年9月，DCloud推进微信开展小程序业务。

- 2016年初，微信团队决定上线小程序业务，但没有接入联盟标准，而是自己制定标准。

- 各巨头无法在标准上达成一致，让开发者面对很多私有标准。

- 为了让标准达成一致，于是有了 uni-app框架，为开发者抹平各平台差异。

- 这就是 uni-app 的由来。

  

## html、vue、小程序的区别

- 传统的 h5 只有1端，即浏览器。而uni-app 可跨多端，虽仍属前端，与传统h5有不同。

- 网络模型变化：以前网页大多是 b/s，现在是 c/s，前后端分享，通过 js api 获取json数据，把数据绑定在界面上渲染。

- 文件类型变化：以前是 .html 文件，开发和运行都是 html。现在是 .vue文件，开发是 vue，经编译，运行时已经变成 js 文件。（现在前端开发，很少直接使用 HTML，基本都是**开发--编译--运行**。）

- 代码架构变化：以前html节点，包含script和style节点。现在 template、script、style 都是1级节点（这叫**vue单文件组件规范SFC**）。

- 外部文件引用方式变化：以前通过 script 的 src、link 的 href 引入外部 js 和 css。现在是 es6 写法，import 引入外部模块（注意不是文件）或css。

- 组件/标签变化：以前是 html 标签，现在是小程序组件 。

- js运行环境变化：运行环境从浏览器变成 v8 引擎。

  - 浏览器运用的 window、document、navigator、location对象、cookie等，**app 和小程序**都**不支持**。

  - js 是 **ECMAScript组织**管理的，浏览器中的js 是 **W3C组织**基于 js 规范补充了 window等专用对象。
  - 在 uni-app，除了 h5端，其他端的 js 都运行在一个独立的 v8 引擎下。
  - 不是在浏览器中，所以浏览器的对象无法使用。
  - 这意味着，依赖document 的很多html库，比如 jquery无法使用。
  - app 和小程序**支持 web-view** 组件，里面可以加载标准的 html。用品退款页面仍然支持浏览器专用对象。

- js数据绑定模式变化：以前的 dom操作，改成vue 的 mvvm。

  - 现在的前端趋势是去 dom 化，改用 mvvm模式，更简洁的写法，大幅减少代码行数，同时**差量渲染**性能更好。

  - uni-app使用vue的数据绑定方式解决js和dom界面交互的问题。

    > 如果你想改变某个dom元素的内容,比如一个 view 的显示文字 :
    >
    > 以前是给 view 设一个 id,然后js里通过选择器获取dom元素,进一步通过js进行赋值操作,修改dom 元素的属性或值.
    >
    > 现在的做法,是vue的绑定模式,给这个dom元素绑定一个js变量,在script中修改js变量的值 ,dom会自动变化,页面会自动更新渲染.

- js api的变化: uni-app的 api是参考小程序的,所以和浏览器的 js api有很多不同.
  - alert,confirm改成uni.showmodel
  - ajax改成 uni.request
  - cookie、session没有了,local.storage改成 uni.storage

- css的变化：**不支持 *** 选择器。没有**body，改为了 page**。px无法动态适应不同宽度的屏幕，rem无法用于nvue/weex。推荐使用 **rpx,全端支持**。推荐使用flex布局，并默认就是flex布局。这个布局思路和传统流式布局有点区别。但flex的有趣在于，不管是什么技术都支持这种排版，web、小程序/快应用、weex/rn、原生的iOS、Android开发，全都支持flex。它是通吃所有端的新一代布局方案。uni-app的vue文件里支持所有web排版方式，不管是流式还是flex。但nvue里，只支持flex，因为它在app端是使用原生排版引擎渲染的。


- 工程结构和页面管理

不需要专门去学习小程序的语法，uni-app使用的是vue的语法，不是小程序自定义的语法。



## uni-app 特色

1. **条件编译**

   

2. App 端的 **Nvue开发**

- Nvue 是 native vue 的简写

- uni-app 提供了一个后缀为 .nuve的文件

- Nvue 内置了 weex 渲染引擎

- weex-->为app提供了原生的渲染能力

  在 Nvue 中，既可以使用 weex 的组件和API，也可以使用uni-app的组件和API。就这相当于在Nvue中，为 weex补充了大量的 uni-app的组件和API.

  

3. App 端的 **Html5 +**

   可以帮我们在 app 中直接调用原生插件，像一些比较复杂的功能，在纯前端很难做到的时候，我们可以使用原生的 android和ios插件来帮助我们实现这些功能。

   同时 html5 + 还提供了几十万的 api 方法。

   

4. **Nvue 和 html5+ 只能在 app 端使用，在 h5和小程序中无法使用**
